# diostatus — The disclose.io Maturity Model

Recognition for early adopters and a clear path towards best practices.

## Let's send best practice viral

diostatus provides recognition to early adopters of vulnerability disclosure programs, and a clear path towards best practices for those wanting to become more advanced.

![diostatus Maturity Model — Level 0 to Level 5](https://disclose.io/uploads/discloseio-maturity-climb-v8.jpg)

## The progression in one line

**Findable → Communicating → Not hostile → Explicitly safe → Accountable.**

Each level builds on the previous, creating a clear progression path for organisations to improve their vulnerability disclosure practices.

## Summary

| Level | Name | Key Signal | Researcher Protection |
|-------|------|------------|----------------------|
| [0](./level-0.md) | Not Present | No contact, no policy | None |
| [1](./level-1.md) | Contact Only | security.txt / intake method exists | None (but reachable) |
| [2](./level-2.md) | Basic VDP | Public policy + channel | None (but documented) |
| [3](./level-3.md) | Partial Safe Harbor | Won't pursue legal action | Partial — report safely |
| [4](./level-4.md) | Full Safe Harbor | Explicitly authorises testing + law exemptions | Full — test safely |
| [5](./level-5.md) | Full Safe Harbor + CVD | Level 4 + proactive disclosure timeline | Full + accountability |

See the individual level pages for plain-English definitions of each.
